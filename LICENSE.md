## 🐛 Common Issues and Fixes

### ❌ macOS says "Macsploit is damaged and can't be opened"
This is a macOS Gatekeeper message — not an actual error with the file. It appears because the app is not from the Mac App Store.

**Fix:** Open **Terminal** and run:
```bash
xattr -cr /Applications/Macsploit.app
```
Then try opening the app again. This removes the macOS quarantine attribute.

---

### ❌ "Injection failed" — no confirmation indicator appears
**Fix:** Ensure Roblox is fully loaded in-server (not on the loading screen) before clicking Inject. Also verify that Accessibility and Screen Recording permissions are both granted in **System Preferences** → Privacy & Security.

---

### ❌ Macsploit not opening after a Roblox update
**Fix:** Open Macsploit and let the auto-updater run. If the auto-updater itself fails, re-download the latest release from this repository. Never run an outdated build after a Roblox client update.

---

### ❌ iOS IPA expires / app disappears after 7 days
**Fix:** This is normal behavior for free Apple Developer certificates. Re-sign the IPA using AltStore (connect to PC/Mac with AltServer running) or switch to Scarlet with a paid certificate for longer validity.

---

### ❌ Scripts execute but produce no visible effect
**Fix:** Many scripts are game-specific. Confirm the script targets the game you are currently playing. Filter the built-in Script Hub by game name to find verified compatible scripts.

---

### ❌ High CPU usage during injection on Intel Mac
**Fix:** Close other resource-heavy applications before launching Macsploit. Intel builds require more CPU for injection than Apple Silicon builds. If the issue persists, check for a newer build that may include additional Intel-specific optimizations.

---

### ❌ macOS Sequoia blocks the app entirely
**Fix:** Go to **System Settings** → Privacy & Security → scroll to the bottom → click **Open Anyway** next to the Macsploit entry. On Sequoia, you may need to do this once after each major Roblox or macOS update.
